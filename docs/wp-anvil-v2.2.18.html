What's new in v2.2.18

Released May 20, 2026 — Web Viewer Parity + Mouse Capture Default-OFF + Autocompact Hardening

Autocompact fix (#697 — most user-visible)

The autocompact threshold was comparing against max_output_tokens (8K–16K) instead of the context window (64K–200K+). On long-context models, sessions were being compacted after as few as 6K tokens of input — roughly one turn's worth. After this fix, sessions correctly run to ~80% of the full context window before compaction fires. If you've been hitting "session was compacted" messages early in long sessions, this is the fix.

Mouse capture default-OFF

Mouse capture is now disabled by default on all platforms. The previous default broke terminal copy-paste for users on Gnome Terminal, kitty, and Windows Terminal who hadn't explicitly opted in. Enable mouse support with /config mouse_capture true or the --mouse flag.

Web viewer tab parity

The AnvilHub web viewer now has the same tab architecture as the desktop TUI: stable tab IDs, /tab new/rename/switch, Ctrl+T, and per-tab routing so messages from one session don't bleed into another. Default layout is now Vertical Split + Tabs. Relay carries cost labels, memory snapshots, and session metadata for accurate status display.

TUI stability fixes

  • Keyboard stops working after /mcp builder cancel — fixed (alt-screen raw-mode restore #688)
  • Bracketed paste in textarea modals — multi-line paste works correctly (#685)
  • MemorySnapshot rail rendering uses shared layout helpers (#695)
  • Release pipeline Phase 6 hardened against silent SSH failures (#654)
