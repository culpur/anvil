# AnvilHub /build Page — Spec v2.2.19

Tasks: #674 (micro-service + page), #675 (security — vault token)

## 1. Overview

The `/build` page extends the Anvil TUI `/mcp builder` wizard (shipped in
v2.2.18, tasks #638 #673 #676 #678 #679 #684) as a web UI on AnvilHub.
Users describe what they want their MCP server to do, get a generated package,
can run it in the sandbox, and publish it to the marketplace.

## 2. Micro-service: anvil-mcp-builder

**Repo:** `/Users/soulofall/projects/mcp-servers/anvil-mcp-builder/`
**Port:** 4090 (localhost only — Apache2 proxies via api.culpur.net)
**Runtime:** Node.js 24 (engines pinned in package.json)

### Endpoints

| Method | Path | Body | Response |
|--------|------|------|----------|
| POST | /api/builder/spec | `{ prompt: string, stream?: boolean }` | `{ data: McpSpec }` or SSE |
| POST | /api/builder/generate | `{ spec: McpSpec }` | `{ data: { tarball (base64), files[], language, specName } }` |
| POST | /api/builder/sandbox | `{ tarball (base64), encoding: 'base64', language, specName }` | `{ data: { install, detonation } }` |
| GET | /health | — | `{ ok: true }` |

### Auth flow

All `/api/builder/*` routes are behind the Authentik forward-auth proxy
(same pattern as all AnvilHub API routes). The proxy injects:

```
X-Forwarded-User:   <email>
X-Forwarded-Email:  <email>
X-Forwarded-Groups: anvilhub-publishers,users
```

The service middleware (`src/auth.js`) reads these headers and attaches
`req.passageUser`. Missing `X-Forwarded-User` returns 401.

Development bypass: `NODE_ENV=development` injects a synthetic `dev@localhost`
user — this is removed in production.

### Rate limiting

`express-rate-limit`: 10 requests per minute per IP, applied before auth.

### Publisher-standing pre-check (#738)

`/api/builder/sandbox` runs `npm install` / `pip install` on user-supplied
tarballs — a real abuse vector. Before extracting the tarball, the handler
calls `hasPublisherStanding(req.passageUser)` (`src/auth.js`). A user passes
the check if EITHER:

1. They are in the `anvilhub-publishers` Authentik group (fast path, no
   network — read from the `X-Forwarded-Groups` header).
2. The AnvilHub backend reports `verified_publisher: true` OR
   `package_count >= 1` via
   `GET ${ANVILHUB_BACKEND_URL}/api/users/<email>/publisher-status`.

Results are cached per email for 5 minutes (`Map<email, {value, expiresAt}>`
in module scope; no external cache). Non-publishers receive
`403 PUBLISHER_REQUIRED` with a hint to publish first or join the group.

`/spec` and `/generate` remain open to any authenticated user — the
publisher pre-check applies only to `/sandbox` per task #737.

Env vars:

| Variable | Default | Notes |
|----------|---------|-------|
| `ANVILHUB_BACKEND_URL` | `https://anvilhub.culpur.net` | Backend used for `/api/users/<email>/publisher-status` lookup |

### Template generation

The `/generate` endpoint re-implements the Rust scaffolding from
`crates/anvil-cli/src/mcp_builder.rs` in JS. The implementation covers:

- Node.js: `package.json` + `index.js` (McpServer + zod) + `test.js` (node:test)
- TypeScript: adds `tsconfig.json`, uses `src/index.ts`, tsx dev dep
- Python: `pyproject.toml` + `__main__.py` (FastMCP) + `test_server.py` (pytest)

All templates emit the honesty-contract return shape (`ok, blockers, warnings,
subresults, message`). No credentials are embedded in generated files.

### Sandbox step

`/sandbox` extracts the tarball to a temp directory, runs `npm install` or
`pip install`, then invokes `anvil-sandbox-runner` without `--allow-network`
(defaults to network-deny on Linux via `unshare -n`, macOS via `sandbox-exec`).
The temp directory is always cleaned up (success or failure).

## 3. Vault key + provisioning (#675)

**Vault key:** `mcpbuilder/operator-oauth-token`
**Vault file:** `~/.anvil/vault/cred_mcpbuilder__operator-oauth-token.enc`
(the vault uses `__` as the path-separator in filenames)

### Provisioning steps (run once on dev0001)

```sh
# Step 1 — ensure vault is initialised
anvil /vault setup          # skip if already done

# Step 2 — store the Anthropic API key
anvil /vault store mcpbuilder/operator-oauth-token
# Anvil will prompt: "Secret value: " — paste the Anthropic API key

# Step 3 — verify
anvil /vault get mcpbuilder/operator-oauth-token
# Should print the key (omit from any log)

# Step 4 — start the service with the vault password in pm2 env
ANVIL_VAULT_PASSWORD='<master-password>' pm2 start ecosystem.config.cjs
```

### Runtime security contract

- The vault is read at service startup via `spawnSync('anvil', ['/vault', 'get', label])`.
- The token is held in process memory only (`_cachedToken` in `src/vault.js`).
- The token is registered with the logger's redact list on load — any log line
  containing the token is replaced with `***REDACTED***` before emission.
- `ANVIL_VAULT_PASSWORD` is passed to the child process via `env:` — not via argv
  (which would appear in `ps aux` output).
- The service refuses to start (exits 1) if the vault password is missing or
  if the vault entry does not exist.
- The generated MCP package NEVER contains the operator token.

### Vault key rotation

```sh
# On vault rotation, update the stored entry and restart the service:
anvil /vault update mcpbuilder/operator-oauth-token
ANVIL_VAULT_PASSWORD='<new-master-password>' pm2 restart anvil-mcp-builder
```

### BuildClient.tsx (client component — three-step wizard)

```tsx
// anvilhub-web/src/app/build/BuildClient.tsx
'use client';

import React, { useState, useRef, useCallback } from 'react';

// ─── Types ────────────────────────────────────────────────────────────────────

type WizardStep = 'prompt' | 'spec-review' | 'build-sandbox';

interface McpSpec {
  name: string;
  language: 'node' | 'typescript' | 'python';
  tools: Array<{
    name: string;
    description: string;
    inputs: Array<{ name: string; description: string; kind: unknown; optional: boolean }>;
  }>;
}

interface GenerateResult {
  tarball: string;
  encoding: 'base64';
  format: 'tar.gz';
  files: string[];
  specName: string;
  language: string;
}

interface SandboxResult {
  install: { ok: boolean; stdout: string; stderr: string };
  detonation: {
    exit_code: number | null;
    stdout_tail: string;
    stderr_tail: string;
    sandbox_backend: string;
    allow_network: boolean;
  } | null;
}

// ─── Step 1: Prompt ───────────────────────────────────────────────────────────

function PromptStep({
  onSpec,
  builderUrl,
}: {
  onSpec: (spec: McpSpec) => void;
  builderUrl: string;
}) {
  const [prompt, setPrompt] = useState('');
  const [streaming, setStreaming] = useState(false);
  const [tokenBuffer, setTokenBuffer] = useState('');
  const [error, setError] = useState<string | null>(null);

  const handleSubmit = useCallback(async () => {
    if (!prompt.trim()) return;
    setStreaming(true);
    setTokenBuffer('');
    setError(null);

    try {
      const res = await fetch(`${builderUrl}/api/builder/spec`, {
        method: 'POST',
        headers: { 'Content-Type': 'application/json', Accept: 'text/event-stream' },
        body: JSON.stringify({ prompt, stream: true }),
      });

      if (!res.ok) {
        const err = await res.json().catch(() => ({}));
        throw new Error(err.error?.message ?? `HTTP ${res.status}`);
      }

      const reader = res.body!.getReader();
      const decoder = new TextDecoder();
      let buffer = '';
      let spec: McpSpec | null = null;

      while (true) {
        const { done, value } = await reader.read();
        if (done) break;
        buffer += decoder.decode(value, { stream: true });

        const lines = buffer.split('\n');
        buffer = lines.pop() ?? '';

        for (const line of lines) {
          if (line.startsWith('event: token')) {
            const dataLine = lines[lines.indexOf(line) + 1] ?? '';
            if (dataLine.startsWith('data: ')) {
              const { text } = JSON.parse(dataLine.slice(6));
              setTokenBuffer((prev) => prev + text);
            }
          } else if (line.startsWith('event: done')) {
            const dataLine = lines[lines.indexOf(line) + 1] ?? '';
            if (dataLine.startsWith('data: ')) {
              const { spec: s } = JSON.parse(dataLine.slice(6));
              spec = s;
            }
          } else if (line.startsWith('event: error')) {
            const dataLine = lines[lines.indexOf(line) + 1] ?? '';
            if (dataLine.startsWith('data: ')) {
              const { message } = JSON.parse(dataLine.slice(6));
              throw new Error(message);
            }
          }
        }
      }

      if (spec) onSpec(spec);
    } catch (err: unknown) {
      setError(err instanceof Error ? err.message : 'Unknown error');
    } finally {
      setStreaming(false);
    }
  }, [prompt, builderUrl, onSpec]);

  return (
    <div className="space-y-4">
      <label className="block text-sm font-medium text-gray-700">
        Describe your MCP server
        <textarea
          className="mt-1 block w-full rounded border-gray-300 shadow-sm text-sm"
          rows={5}
          placeholder="e.g. A weather tool that takes a city name and returns the current temperature and conditions"
          value={prompt}
          onChange={(e) => setPrompt(e.target.value)}
          disabled={streaming}
        />
      </label>

      {streaming && tokenBuffer && (
        <pre className="text-xs text-gray-500 whitespace-pre-wrap font-mono bg-gray-50 p-2 rounded border max-h-48 overflow-auto">
          {tokenBuffer}
        </pre>
      )}

      {error && (
        <p className="text-sm text-red-600">Error: {error}</p>
      )}

      <button
        onClick={handleSubmit}
        disabled={streaming || !prompt.trim()}
        className="px-4 py-2 bg-blue-600 text-white rounded text-sm font-medium disabled:opacity-50"
      >
        {streaming ? 'Generating spec...' : 'Generate spec'}
      </button>
    </div>
  );
}

// ─── Step 2: Spec review ──────────────────────────────────────────────────────

function SpecReviewStep({
  spec,
  onEdit,
  onConfirm,
}: {
  spec: McpSpec;
  onEdit: (spec: McpSpec) => void;
  onConfirm: () => void;
}) {
  return (
    <div className="space-y-4">
      <div className="bg-gray-50 border rounded p-4 space-y-2 text-sm">
        <div><span className="font-medium">Name:</span> {spec.name}</div>
        <div><span className="font-medium">Language:</span> {spec.language}</div>
        <div>
          <span className="font-medium">Tools ({spec.tools.length}):</span>
          <ul className="mt-1 ml-4 list-disc space-y-1">
            {spec.tools.map((t) => (
              <li key={t.name}>
                <span className="font-mono">{t.name}</span> — {t.description}
                {t.inputs.length > 0 && (
                  <span className="text-gray-400"> ({t.inputs.length} input{t.inputs.length !== 1 ? 's' : ''})</span>
                )}
              </li>
            ))}
          </ul>
        </div>
      </div>

      <div className="flex gap-3">
        <button
          onClick={onConfirm}
          className="px-4 py-2 bg-blue-600 text-white rounded text-sm font-medium"
        >
          Generate package
        </button>
        <button
          onClick={() => onEdit(spec)}
          className="px-4 py-2 border rounded text-sm font-medium text-gray-600"
        >
          Edit JSON
        </button>
      </div>
    </div>
  );
}

// ─── Step 3: Generate + Sandbox + Publish ─────────────────────────────────────

function BuildSandboxStep({
  spec,
  builderUrl,
}: {
  spec: McpSpec;
  builderUrl: string;
}) {
  const [phase, setPhase] = useState<'idle' | 'generating' | 'sandboxing' | 'done' | 'error'>('idle');
  const [generateResult, setGenerateResult] = useState<GenerateResult | null>(null);
  const [sandboxResult, setSandboxResult] = useState<SandboxResult | null>(null);
  const [error, setError] = useState<string | null>(null);

  const run = useCallback(async () => {
    setPhase('generating');
    setError(null);

    // Step A: generate package
    let genData: GenerateResult;
    try {
      const res = await fetch(`${builderUrl}/api/builder/generate`, {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ spec }),
      });
      const json = await res.json();
      if (!res.ok) throw new Error(json.error?.message ?? `HTTP ${res.status}`);
      genData = json.data;
      setGenerateResult(genData);
    } catch (err: unknown) {
      setError(err instanceof Error ? err.message : 'Generation failed');
      setPhase('error');
      return;
    }

    // Step B: sandbox
    setPhase('sandboxing');
    try {
      const res = await fetch(`${builderUrl}/api/builder/sandbox`, {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({
          tarball: genData.tarball,
          encoding: 'base64',
          language: genData.language,
          specName: genData.specName,
        }),
      });
      const json = await res.json();
      if (!res.ok) throw new Error(json.error?.message ?? `HTTP ${res.status}`);
      setSandboxResult(json.data);
      setPhase('done');
    } catch (err: unknown) {
      setError(err instanceof Error ? err.message : 'Sandbox failed');
      setPhase('error');
    }
  }, [spec, builderUrl]);

  return (
    <div className="space-y-4">
      {phase === 'idle' && (
        <button
          onClick={run}
          className="px-4 py-2 bg-blue-600 text-white rounded text-sm font-medium"
        >
          Build and test in sandbox
        </button>
      )}

      {phase === 'generating' && (
        <p className="text-sm text-gray-500">Generating package...</p>
      )}

      {phase === 'sandboxing' && generateResult && (
        <div className="space-y-2">
          <p className="text-sm text-green-600">Package generated ({generateResult.files.length} files). Running sandbox...</p>
          <ul className="text-xs text-gray-400 font-mono list-none ml-2">
            {generateResult.files.map((f) => <li key={f}>{f}</li>)}
          </ul>
        </div>
      )}

      {phase === 'done' && sandboxResult && (
        <div className="space-y-3">
          <div className={`text-sm font-medium ${sandboxResult.install.ok ? 'text-green-600' : 'text-red-600'}`}>
            Install: {sandboxResult.install.ok ? 'OK' : 'Failed'}
          </div>
          {sandboxResult.detonation && (
            <div className="text-sm">
              <span className="font-medium">Sandbox:</span>{' '}
              exit code {sandboxResult.detonation.exit_code ?? 'killed'}{' '}
              via {sandboxResult.detonation.sandbox_backend}{' '}
              (network: {sandboxResult.detonation.allow_network ? 'allowed' : 'blocked'})
            </div>
          )}
          {sandboxResult.detonation?.stdout_tail && (
            <pre className="text-xs font-mono bg-gray-50 border rounded p-2 max-h-32 overflow-auto">
              {sandboxResult.detonation.stdout_tail}
            </pre>
          )}
          <div className="pt-2">
            <p className="text-xs text-gray-400 mb-2">
              To publish to the marketplace, use the Anvil CLI: <span className="font-mono">anvil /mcp publish</span>
            </p>
          </div>
        </div>
      )}

      {phase === 'error' && (
        <p className="text-sm text-red-600">Error: {error}</p>
      )}
    </div>
  );
}

// ─── Wizard shell ─────────────────────────────────────────────────────────────

export function BuildClient({ builderUrl }: { builderUrl: string }) {
  const [step, setStep] = useState<WizardStep>('prompt');
  const [spec, setSpec] = useState<McpSpec | null>(null);

  const steps: Array<{ id: WizardStep; label: string }> = [
    { id: 'prompt', label: '1. Describe' },
    { id: 'spec-review', label: '2. Review spec' },
    { id: 'build-sandbox', label: '3. Build & test' },
  ];

  return (
    <div className="space-y-6">
      {/* Progress indicator */}
      <nav className="flex gap-4 text-sm" aria-label="Build wizard steps">
        {steps.map((s) => (
          <span
            key={s.id}
            className={step === s.id ? 'font-semibold text-blue-700' : 'text-gray-400'}
          >
            {s.label}
          </span>
        ))}
      </nav>

      {/* Step content */}
      {step === 'prompt' && (
        <PromptStep
          builderUrl={builderUrl}
          onSpec={(s) => {
            setSpec(s);
            setStep('spec-review');
          }}
        />
      )}

      {step === 'spec-review' && spec && (
        <SpecReviewStep
          spec={spec}
          onEdit={(edited) => setSpec(edited)}
          onConfirm={() => setStep('build-sandbox')}
        />
      )}

      {step === 'build-sandbox' && spec && (
        <BuildSandboxStep spec={spec} builderUrl={builderUrl} />
      )}
    </div>
  );
}
```

## 7. Open questions

1. **Passage scope for Build page users** — currently any authenticated AnvilHub
   user can call `/api/builder/*`. Should the `authorize(['anvilhub-publishers'])` 
   middleware be added to all three routes (not just publish)? The `authorize` 
   helper in `src/auth.js` is ready; it just needs to be mounted in `src/server.js`.

2. **Hub publisher pre-check on /sandbox and /generate** — should we require a
   verified publisher account before allowing sandbox runs (to prevent abuse of
   the install step)? The Authentik groups header makes this a one-liner.

3. **Tarball size limit** — the current body parser limit is 64 KB. A Python package
   with heavy pip deps post-install could exceed this on the sandbox response.
   Consider bumping to 512 KB for the sandbox route body.

4. **anvil-sandbox-runner binary on dev0001** — the sandbox step silently degrades
   if `anvil-sandbox-runner` is not on PATH (returns a `sandbox_backend: unavailable`
   report). Need to verify the binary is in `/usr/local/bin` on dev0001 or adjust
   PATH in ecosystem.config.cjs.

5. **Streaming SSE and Authentik proxy** — the Authentik forward-auth proxy may
   buffer SSE responses. Verify `ProxyPass` has `flushpackets=on` or add
   `Header set X-Accel-Buffering "no"` to the Apache2 VirtualHost for the `/spec`
   route.
