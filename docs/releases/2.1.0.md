# Anvil v2.1.0 Release Notes

**Release Date:** April 8, 2026

---

## Headline: Security-First Architecture

Anvil v2.1.0 is a security-focused release that encrypts all credentials, sandboxes file operations, and restructures the entire codebase into a modular architecture. 394 tests pass with zero warnings.

---

## New Features

### Encrypted Credential Vault

All API keys, tokens, and credentials are now stored in an **AES-256-GCM encrypted vault** instead of plaintext JSON files.

- **Setup wizard integration** — the first-run wizard now prompts for a vault master password before any provider configuration. Every API key entered during setup is encrypted immediately.
- **Argon2id key derivation** — 65MB memory cost, 3 iterations, 4-way parallelism. Resistant to GPU and ASIC attacks.
- **Envelope encryption** — each credential gets its own random 256-bit Data Encryption Key (DEK), encrypted under the master Key Encryption Key (KEK). Compromising one credential does not expose others.
- **Session-based unlock** — password prompted once per session. The KEK is held in memory and zeroed on exit.
- **Migration path** — existing users with plaintext `credentials.json` are prompted to migrate. The old file is renamed to `.bak` after successful migration.
- **Backward compatible** — env vars still override vault credentials. Non-interactive modes (CI/CD) skip prompts and use env vars directly.

### File Write Sandbox

Agents can no longer write outside your project boundary.

- **Project root detection** — walks up from CWD looking for `.git`, `.anvil`, `Cargo.toml`, `package.json`
- **Write/edit blocked** outside project root with `PermissionDenied` error
- **Always-allowed targets** — `/tmp`, `$TMPDIR`, `~/.anvil/` are never blocked
- **Power user bypass** — `ANVIL_ALLOW_GLOBAL_WRITES=1` disables the sandbox
- **Read operations** — allowed everywhere with out-of-boundary warnings to stderr
- **Search scoping** — `glob_search` and `grep_search` default to project root

### Native Ollama API

Replaced OpenAI-compatible endpoint with Ollama's native `/api/chat`:

- **Proper thinking mode** — `think: true/false` parameter for qwen3, qwq, deepseek-r1, deepseek-v3
- **NDJSON streaming** — native newline-delimited JSON instead of SSE parsing
- **Token tracking** — `prompt_eval_count`/`eval_count` from final chunk
- **Thinking blocks** — mapped to `ContentBlockStart(Thinking)` + `ThinkingDelta` events
- **OpenAI-compat fallback** retained for explicit use

### Multi-Line Input

The TUI input area now dynamically expands from 1 to 5 lines based on content:

- Soft word-wrap at terminal width boundaries
- Literal newlines via Ctrl+J
- Cursor tracks correctly across wrapped rows
- Footer height adjusts dynamically (content area shrinks)

### Ctrl+C Behavior (Claude Code Parity)

- **First Ctrl+C** — clears input line
- **Second Ctrl+C** (within 1 second, empty input) — exits Anvil
- Shows "(press Ctrl+C again to exit)" hint

### Content Filter Updates

- Added detection for modern OpenAI API key formats: `sk-proj-*`, `sk-svcacct-*`
- Existing patterns for AWS, GitHub, Anthropic, Stripe, Slack preserved

## Upgrade Guide

### From v2.0.0

1. Update: `brew upgrade anvil` or `anvil --update`
2. On first launch, Anvil will prompt you to set a vault password
3. Existing credentials in `credentials.json` will be migrated to the vault
4. The old `credentials.json` is renamed to `credentials.json.bak`
5. Environment variables continue to work as before

### From v1.x

1. Update to v2.1.0 using your preferred method
2. Run `anvil` — the setup wizard will guide you through vault setup and provider configuration
3. All features from v1.x are preserved

---

## Security Advisory

- **credentials.json deprecation** — plaintext credential storage is now deprecated. While still supported as a fallback, all new installations use the encrypted vault. Existing users are prompted to migrate on first launch.
- **File write sandbox** — agents are now sandboxed to the project root by default. If you have workflows that write outside the project, set `ANVIL_ALLOW_GLOBAL_WRITES=1`.
