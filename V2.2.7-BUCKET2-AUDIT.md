# v2.2.7 Bucket 2 — Cross-Platform Env Handling Audit

## Scope

Fix two startup bugs on Windows 11 and improve portability across the Anvil
workspace:

- **Bug A** — Anvil fails at startup with HOME/PATH errors on Windows.
  `HOME` is a Unix-only env var; Windows uses `USERPROFILE`.
- **Bug B** — QMD binary not found on Windows even when installed.
  The detection code used `Command::new("which")` (Unix-only) and
  hardcoded `/opt/homebrew/bin/qmd`.

### crates/runtime/src/qmd.rs — Bug B core fix

**Before:**
```rust
fn detect_binary() -> Option<PathBuf> {
    let output = Command::new("which").arg("qmd").output().ok()?;
    if !output.status.success() { return None; }
    let path_str = String::from_utf8(output.stdout).ok()?;
    Some(PathBuf::from(path_str.trim()))
}
```

**After:**
```rust
fn detect_binary() -> Option<PathBuf> {
    if let Ok(p) = which::which("qmd") { return Some(p); }
    #[cfg(target_os = "macos")] {
        let homebrew = PathBuf::from("/opt/homebrew/bin/qmd");
        if homebrew.is_file() { return Some(homebrew); }
    }
    #[cfg(target_os = "windows")] {
        for candidate in [
            std::env::var("LOCALAPPDATA").ok()
                .map(|d| PathBuf::from(d).join("Programs").join("qmd").join("qmd.exe")),
            std::env::var("PROGRAMFILES").ok()
                .map(|d| PathBuf::from(d).join("qmd").join("qmd.exe")),
        ].into_iter().flatten() {
            if candidate.is_file() { return Some(candidate); }
        }
    }
    None
}
```

The `which` crate handles `PATHEXT` on Windows automatically.

### crates/anvil-cli/src/utils.rs — Bug A core fix

**Before:**
```rust
pub(crate) fn command_exists(name: &str) -> bool {
    Command::new("which").arg(name).output()
        .map(|o| o.status.success()).unwrap_or(false)
}
```

**After:**
```rust
pub(crate) fn command_exists(name: &str) -> bool {
    let path_os = match std::env::var_os("PATH") { Some(v) => v, None => return false };
    for dir in std::env::split_paths(&path_os) {
        if dir.join(name).is_file() { return true; }
        #[cfg(target_os = "windows")]
        if let Ok(pathext) = std::env::var("PATHEXT") {
            for ext in pathext.split(';') {
                if dir.join(format!("{name}{ext}")).is_file() { return true; }
            }
        }
    }
    false
}
```

Home dir: `env::var("HOME")` → `dirs_next::home_dir()`.

## Backwards Compatibility

The backwards-compat rule is preserved throughout: all code continues to use
`~/.anvil` as the config directory. The only change is *how* `~` is resolved —
using `dirs_next::home_dir()` instead of `env::var("HOME")`. On macOS/Linux
`dirs_next::home_dir()` returns `$HOME`, so existing installs are unaffected.
