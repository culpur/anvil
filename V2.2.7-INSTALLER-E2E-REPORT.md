## Additional Defect: B-1 — Stale variable `${SHA256_URL}` in install.sh (line 202)

install.sh sets `set -euo pipefail` at line 13. The `-u` flag causes any reference to an unset variable to be a fatal error. Line 202 references `${SHA256_URL}`, which is never defined — only `SHA256_URL_PRIMARY` and `SHA256_URL_FALLBACK` exist.

**Trigger path:** checksum file is fetched successfully by curl (so the download/fallback code does not run) but the file content is empty or contains only whitespace. This triggers the `[[ -z "${EXPECTED}" ]]` guard at line 201, which then tries to run line 202.

**Observed behaviour (confirmed in Docker):**
```
/tmp/install-test.sh: line 202: SHA256_URL: unbound variable
INSTALL_EXIT:1
```

**Expected behaviour:** Script should exit with code 2 and message: `"Checksum file at ... is empty or malformed."` (the message at line 202 is correct but the variable reference is wrong).

**Impact:**
- Wrong exit code: `1` instead of the specified `2` for checksum failures.
- Misleading error: `SHA256_URL: unbound variable` instead of the checksum-integrity message.
- Does NOT bypass security — install still aborts.

The identical bug exists in `install.ps1` line 157 (`$Sha256Url` instead of `$Sha256UrlPrimary`/`$Sha256UrlFallback`), but PowerShell's `Set-StrictMode -Version Latest` would raise a runtime error on that line.

## Root Cause of T5/T6/T7/T8 Failures: Release Artifacts Predate v2.2.7 Feature Commits

The binaries in `target/release-artifacts/` (all four platforms) were cross-compiled at **18:31** on 2026-04-21. The v2.2.7 feature commits that introduced `check.rs`, `setup.rs`, `upgrade.rs`, and the new Rust CLI arg parsing landed at **21:48–22:02** the same day. The security-fix commits (ba09290, c68389b) landed at **22:01–22:02**.

Consequence: the binaries in `target/release-artifacts/` are compiled from commit `5971391` (the v2.2.6 release commit, which bumped the version string but did not yet include the new CLI code). The binaries are a Node.js-bundled agent (Claude Code variant) that does **not** implement:

- `--check` → dispatches to Rust `run_check()` / `check.rs`
- `--uninstall` → dispatches to `run_uninstall()` / `upgrade.rs`
- `--setup` with `rpassword` no-echo + `chmod 0600` → `setup.rs`
- `--upgrade` with out-of-band SHA256 + abort-on-failure → `upgrade.rs`

The C-3 security fix (rpassword + 0600) is in the **source** (setup.rs, committed in ba09290) but is **not in the distributed binary**. A new build is required.

## RELEASE SAFE? **NO**

### Blockers before release may fire:

1. **B-3 (Critical):** The binary release artifacts must be rebuilt from HEAD (`c68389b`) before upload. The current binaries do not contain the C-3 security fix (setup.rs: rpassword no-echo, chmod 0600 on config.json). They also do not implement `--check`, `--uninstall`, or `--upgrade` as Rust CLI actions.

2. **B-1 (Medium):** `install/install.sh` line 202 — replace `${SHA256_URL}` with `${SHA256_URL_PRIMARY:-unknown}` (or the applicable source URL). This crashes with a confusing error message under `set -u` when a checksum file is empty; exit code is 1 not 2, breaking the exit-code contract.

3. **B-2 (Medium):** `install/install.ps1` line 157 — replace `$Sha256Url` with the appropriate `$Sha256UrlPrimary` or `$Sha256UrlFallback` variable. Undefined variable under `Set-StrictMode Latest`.

### Conditional items (can ship if accepted):

4. **B-4 (Low):** T9 spec requires `/v` prefix completions. None of the completion files implement slash-command completions. Accept as out-of-scope for v2.2.7, or add before shipping.
