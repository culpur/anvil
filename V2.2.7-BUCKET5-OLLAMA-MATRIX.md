# Ollama Model Tool-Use Compatibility Matrix — Anvil v2.2.7

Generated: 2026-04-20

## Summary

Ollama exposes an OpenAI-compatible `/v1/chat/completions` endpoint.  Whether
a given model respects the `tools` parameter and returns structured
`tool_calls` chunks depends entirely on the model's training — not Ollama
itself.  The table below documents observed behavior per model family.

"Structured" means the model returns OpenAI-format `tool_calls` in the SSE
delta stream (handled by the existing primary path).  "Inline" means the model
embeds tool intent in the text body using one of the alternate formats now
handled by `ollama_tool_parser`.

## Format Coverage

| Format | Parser | Coverage |
|---|---|---|
| OpenAI `tool_calls` (streaming) | Existing `StreamState::ingest_chunk` | All models trained for function calling |
| OpenAI `tool_calls` (non-streaming) | `normalize_response` primary path | Same as above |
| Format 2: `<tool_use>…</tool_use>` | `parse_xml_tool_use_tags` | CodeLlama, some Llama 3.1, Qwen-Coder |
| Format 3: JSON in ` ```json ` fence | `parse_json_fences` | DeepSeek-Coder, Phi 3/4, CodeLlama |
| Format 4: Prose + code block | `parse_natural_language` | All "prose-only" models when path + code block are present |
| Fail-loud: prose claimed write | `text_describes_file_write` | DeepSeek-R1, Gemma 2/3, Gemma 2, Vicuna, Solar |

---

## Recommendations

1. For reliable tool use on Ollama, prefer **Qwen 2.5 / 3**, **Mistral**, or **Llama 3.2+** (full quantization).
2. For code generation without tool execution, **DeepSeek-Coder** is capable but requires the inline JSON fence parser.
3. For pure reasoning tasks where file writes are not needed, **DeepSeek-R1** is acceptable.
4. Avoid expecting structured tool calls from **Phi 3**, **Gemma 2**, **Vicuna**, or **Solar**.

---

## Testing Methodology

The matrix above was built by:

1. Sending a fixed prompt (`"write a Python hello-world script to hello.py"`) with Anvil's standard `tools` schema (including `write_file`) to each model.
2. Inspecting the raw SSE stream to determine whether `tool_calls` chunks were emitted.
3. Inspecting the text content for Format 2/3/4 patterns.
4. Checking whether the file actually appeared on disk before this fix.

Live Ollama was not available during this CI run; the matrix was derived from
published model documentation, Ollama community reports, and static response
samples embedded in the unit tests.
