## How to host on anvilhub.culpur.net

**Immediate path** — copy to AnvilHub's `public/` directory:

```sh
cp install/install.sh   /path/to/anvilhub-web/public/
cp install/install.ps1  /path/to/anvilhub-web/public/
cp -r install/completions /path/to/anvilhub-web/public/
```

Then rebuild and restart:

```sh
npx next build
pm2 restart anvilhub-web
```

The scripts will be served at:
- `https://anvilhub.culpur.net/install.sh`
- `https://anvilhub.culpur.net/install.ps1`

## How to build .deb and .rpm

Full instructions in `install/packaging.md`. Summary:

```sh
cargo install cargo-deb cargo-generate-rpm
cargo build --release -p anvil-cli
cargo deb -p anvil-cli
cargo generate-rpm -p crates/anvil-cli
```

Add the `[package.metadata.deb]` / `[package.metadata.generate-rpm]` blocks to `crates/anvil-cli/Cargo.toml` as shown in `install/packaging.md` before running.

## anvil --check behaviour

Prints a green/red checklist with 8 rows:

1. Anvil on PATH
2. Node.js (warns if < v18)
3. npm
4. Git
5. QMD (searches PATH + common locations)
6. Vault status (checks `~/.anvil/vault/` + `config.json`)
7. API keys (inspects `config.json` for any configured provider)
8. Relay reachable (HTTP ping to `passage.culpur.net`)

Exit code: 0 if all pass, 1 if any fail.

## anvil upgrade behaviour

1. Fetches latest tag from `api.github.com/repos/culpur/anvil/releases/latest`
2. Compares against `CARGO_PKG_VERSION`
3. Downloads `anvil-<target>` from GitHub Releases
4. Fetches `anvil-<target>.sha256` and verifies (aborts with exit 2 on mismatch)
5. Atomic rename: writes `.new`, renames current to `.bak`, renames `.new` to current, removes `.bak`
6. Falls back to `/tmp/anvil-new` with manual instructions if permissions fail
7. Respawns using `respawn::respawn()` so the terminal session survives
